The AI Agent Control Landscape — March 2026

March 2026 brought major advances in AI agent control — new frameworks, new security stacks, new funding rounds. But it also revealed a massive gap that none of them fill. Here's every major tool in the stack, what layer they operate at, what they do well, and why execution authority over real business systems remains unsolved.

Five Layers, One Missing

As AI agents move from advisory to autonomous, an entire ecosystem of tools has emerged to make them safer, more compliant, and more controlled. These tools are real, well-funded, and solve genuine problems.

But they all share one characteristic: the agent still executes on its own.

Every tool in this landscape operates on the agent's inputs, outputs, model behavior, access permissions, or runtime environment. None of them sit in the execution path between the agent and the business systems it touches. None of them sit in the execution path. None of them evaluate whether a specific action should happen for the business right now.

This post maps every major player, the layer they occupy, and the gap that remains.

Layer 1: Output Validation

Answers: "Is this output safe?"

Guardrails AI
Output Validation
Founded 2023 $7.5M Seed (Feb 2024) ~$1.1M revenue
Open-source Python framework for validating LLM text inputs and outputs. Checks for toxicity, PII leakage, hallucinations, schema compliance, and custom validation rules. Includes Guardrails Hub — a community marketplace of reusable validators — and Snowglobe, a simulation engine for testing guardrails before production.
guardrailsai.com GitHub Guardrails OSS Snowglobe
Backed by Zetta Venture Partners, Bloomberg Beta, Pear VC, GitHub Fund. Angels include Ian Goodfellow and Logan Kilpatrick. 11 employees.
The gap: Guardrails validates what the model says — not what the agent does with it. It runs inside the agent's pipeline as a Python library. The agent still holds the credentials. The agent can bypass, remove, or reconfigure the guardrail. Validation is advisory. The agent still executes on its own.

Also in this layer:

NVIDIA NeMo Guardrails — NVIDIA's programmable guardrails framework for LLM applications. Defines conversational boundaries using Colang, a domain-specific language. Controls what the model can say and how it responds, but does not govern agent actions on external systems. GitHub

Layer 2: Model Behavior Control

Answers: "Is this model compliant?"

CTGT
Model Behavior
Founded 2023 $7.7M Seed (Feb 2025) YC W24
Modifies model behavior at the representation level using mechanistic interpretability. Identifies and removes hallucinations, bias, and unwanted behavior in LLMs without retraining or fine-tuning. Claims 500x faster model deployment. Their product Mentat provides a runtime policy engine that enforces compliance at the inference layer.
ctgt.ai Mentat Platform Access
Backed by Google's Gradient, General Catalyst, Y Combinator, Liquid 2, Deepwater. Angels include François Chollet, Paul Graham, Michael Seibel. Working with Fortune 10 companies including PwC and JP Morgan.
The gap: CTGT controls what the model produces — not what the agent does with that output. It operates at the inference layer, shaping model behavior before the agent even acts. It does not sit between the agent and external systems. The agent still holds the credentials. The agent still executes on its own.

Layer 3: Sandbox & Environment Security

Answers: "Can this agent access this?"

NVIDIA NemoClaw + OpenShell
Sandbox & Access
Alpha: March 16, 2026 GTC 2026 launch Open source
Reference stack for running OpenClaw agents inside a secure sandbox. Built on NVIDIA OpenShell runtime with Rust-based container isolation, network egress control, filesystem restrictions, and declarative YAML security policies. Includes managed inference with NVIDIA Nemotron models. Announced at GTC 2026. Not production-ready — APIs and behavior may change.
GitHub nvidia.com/nemoclaw Developer Guide Press Release
NVIDIA is partnering with Cisco, CrowdStrike, Google, and Microsoft Security to integrate OpenShell into enterprise security stacks. Jensen Huang called OpenClaw "the operating system for personal AI."
The gap: NemoClaw controls WHERE the agent can go — which network endpoints it can reach, which files it can access. Once access is granted (because the agent needs GitHub access to be useful), NemoClaw has no opinion on whether THIS SPECIFIC PR merge should happen. It checks the door. It does not evaluate the action. The agent still executes on its own.

Also in this layer:

IronClaw — Container-level security for OpenClaw agents. Rust-based network isolation and sandboxing. Part of the broader OpenShell ecosystem. Controls access, not business decisions.

Layer 4: Infrastructure & Endpoint Security

Answers: "Is this process allowed to run?"

CrowdStrike · Palo Alto Networks · Okta
Infrastructure Security
Public companies Established players
Enterprise-grade infrastructure, endpoint, and identity security. CrowdStrike detects and prevents threats at the endpoint level. Palo Alto Networks secures network traffic and cloud infrastructure. Okta manages identity and access. These are the foundation of enterprise security — but they operate below the application layer.
crowdstrike.com paloaltonetworks.com okta.com
The gap: Infrastructure security answers "is this process allowed to run on this machine?" It has no concept of what the agent is doing at the application layer. CrowdStrike doesn't know the difference between a Slack message to #eng-platform and a Slack message to #company-announcements. It secures the infrastructure. The agent still executes on its own.

Layer 5: Agent Runtimes & Frameworks

Answers: "What should the agent do?"

OpenClaw · LangChain · CrewAI · AutoGen
Agent Runtime
OpenClaw: 250K+ GitHub stars LangChain: $125M Series A (Oct 2025)
Agent frameworks that plan, reason, and execute. OpenClaw is the fastest-growing open source agent runtime, described by NVIDIA as "the operating system for personal AI." LangChain provides the most widely-used tool/chain orchestration. CrewAI enables multi-agent collaboration. AutoGen (Microsoft) enables multi-agent conversation patterns.
OpenClaw GitHub langchain.com crewai.com AutoGen GitHub
Some runtimes are adding internal governance features — approval workflows, tool allowlists, permission systems. OpenClaw's ClawHub marketplace now includes security-focused plugins.
The gap: When governance lives inside the agent framework, it's controlled by the same system it's supposed to constrain. The agent still holds the credentials. A model update, a prompt change, or a framework reconfiguration can silently alter what's "allowed." Self-regulation is not enforcement. The agent still executes on its own.

Related: MCP (Model Context Protocol)

Model Context Protocol — Anthropic's open standard for connecting AI models to external data sources and tools. MCP standardizes how agents connect to systems, but does not evaluate or govern what the agent does once connected. It's a transport layer, not a decision layer. GitHub

The Missing Layer: Execution Authority

Layer 6 — Execution Authority
Surfit
WHERE EXECUTION HAPPENS

Every tool above feeds into the agent. Surfit is where the agent's output becomes real.

Surfit sits between the agent and every system it touches — Slack, GitHub, X, Notion, Gmail, Outlook, AWS, and more. The agent proposes an action. Surfit evaluates it in business context. Low-risk actions execute instantly. High-risk actions are held and routed. Everything produces a receipt.
