1. Overview

Surfit ("we," "our," "us") provides a decision layer for AI agent actions. This Privacy Policy explains what data we collect, how we use it, how we protect it, and your rights regarding that data. We are committed to protecting the privacy and security of our customers and their data.

2. Data We Collect

2.1 Customer Account Data

When you create a Surfit account, we collect:

  • Organization name and contact information
  • Email address for account administration and notifications
  • Dashboard login credentials (stored as salted hashes, never in plaintext)

2.2 System Credentials

To execute governed actions on your behalf, Surfit stores credentials for your connected systems (Slack, GitHub, X, Notion, Gmail, Outlook, AWS, and others).

  • Credentials are encrypted using AES-256 with per-tenant derived keys (HKDF)
  • The master encryption key is stored as a restricted environment variable, never in code or database
  • Credentials are decrypted only at execution time, inside the function that makes the API call, and immediately discarded
  • Credentials never appear in logs, dashboards, API responses, or error messages
  • You can revoke any credential at any time — revocation is immediate and permanent

When your agents send actions through Surfit, we process and store:

2.4 Technical Data

We collect standard technical data for service operation: IP addresses in server logs, API request metadata, and error logs. This data is used for debugging, security monitoring, and rate limiting. Server logs are retained for 30 days.

3. How We Use Your Data

  • Service delivery: Evaluating agent actions, executing approved actions, generating receipts and compliance reports
  • Security: Credential access auditing, threat detection, anomaly detection, rate limiting
  • Service improvement: Aggregate, anonymized usage patterns to improve Wave classification accuracy

We do not:

  • Sell, rent, or share your data with third parties
  • Use your data to train machine learning models
  • Access your system credentials for any purpose beyond executing governed actions
  • Display credential values in any dashboard, log, API response, or report

4. Data Isolation

Surfit uses per-tenant data isolation:

  • Each customer's data is stored in tenant-scoped database records
  • Encryption keys are derived per-tenant — compromising one tenant's key does not expose another's credentials
  • API endpoints enforce tenant scoping via authenticated customer identifiers
  • The operator (Surfit) has cross-tenant visibility for administrative purposes only

5. Data Security

  • Encryption in transit: All connections use TLS 1.2 or higher. HTTP requests are redirected to HTTPS.
  • Encryption at rest: System credentials are AES-256 encrypted with per-tenant derived keys.
  • Access control: Administrative operations require two-factor authentication (TOTP). API endpoints require API key authentication.
  • Audit logging: Every credential access and administrative action is recorded in a hash-chained, tamper-evident audit log. Any modification to the log breaks the chain and is cryptographically detectable.
  • Rate limiting: API endpoints are rate-limited to prevent abuse (100 requests per minute per tenant).
  • Emergency controls: Global kill switch halts all execution immediately. Per-tenant Lagoon mode enables evaluation without execution.

6. Data Retention and Deletion

  • Action records, audit logs, and agent intelligence data are retained for 12 months from creation.
  • System credentials are retained until revoked by the customer. Revocation is immediate and permanent (hard delete, not soft delete).
  • Upon account termination, all customer data (actions, credentials, agent data, audit logs) is deleted within 30 days upon written request to support@surfit.ai.
  • Backup copies are retained for 14 days after deletion and then permanently removed.

7. Third-Party Services

Surfit interacts with third-party systems (Slack, GitHub, X, Notion, Gmail, Outlook, AWS) solely to execute governed actions on behalf of the customer using customer-provided credentials. Surfit does not share customer data with these services beyond the action being executed. Each third-party service is subject to its own privacy policy and terms.

8. International Data Transfers

Surfit's infrastructure is hosted in the European Union (Germany). Customer data is processed and stored within the EU. If you are located outside the EU, your data is transferred to and processed in the EU.

9. Your Rights

You have the right to:

  • Access: Request a copy of your data at any time via compliance report export or by contacting support@surfit.ai
  • Rectification: Request correction of inaccurate data
  • Deletion: Request deletion of your data (subject to retention requirements)
  • Portability: Export your action data and compliance reports
  • Revocation: Revoke any or all system credentials immediately
  • Restriction: Request that Surfit limit processing of your data

To exercise any of these rights, contact support@surfit.ai.

10. GDPR Compliance

For customers subject to the General Data Protection Regulation (GDPR):

  • Legal basis: We process your data based on the contractual necessity of providing the Service (Article 6(1)(b)).
  • Data controller: The customer is the data controller for action content processed through Surfit. Surfit acts as a data processor.
  • Data Processing Agreement: Available upon request for enterprise customers.
  • Data Protection Officer: Contact support@surfit.ai for data protection inquiries.

11. Children's Privacy

Surfit is a business-to-business service and is not intended for use by individuals under 18 years of age. We do not knowingly collect data from children.

12. Incident Notification

In the event of a data breach affecting customer data:

  • Affected customers will be notified within 72 hours of discovery
  • For credential exposure incidents (P1), the kill switch is activated immediately, affected credentials are revoked, and customers are notified within 4 hours
  • Forensic analysis is conducted via hash-chained audit logs to determine scope
  • Applicable regulatory authorities will be notified as required by law

13. Changes to This Policy

We may update this Privacy Policy with 30 days notice to active customers. Material changes will be communicated via email or dashboard notification. The "Last Updated" date at the top reflects the most recent revision.
